# 📘 Learnings

---

## 🧠 Technical Learnings

- Handling **imbalanced datasets** is critical in real-world ML problems
- Accuracy is not a reliable metric in fraud detection
- Ensemble models like Random Forest perform well on structured data
- Anomaly detection models help detect rare events

---

## 🛠️ Engineering Learnings

- Writing modular code improves scalability and readability
- Separating logic into `src/` improves maintainability
- Saving models (`.pkl`) enables reuse and deployment
- Documentation is as important as implementation

---

## 📊 Data Science Insights

- Fraud patterns are often subtle and non-linear
- Feature scaling significantly impacts model performance
- Model evaluation must align with business goals (recall > accuracy)
